# Clear dataset

`data/derived/clear/` is the no-obstacle image pool used as the clear class in
[[kaggle-two-head-dataset]]. Its training label is:

```text
obstacle_class = 0
deviation_class = 2
```

## Current state

As of 2026-06-11:

- `data/derived/clear/images/` contains **1136 JPGs**.
- `data/derived/clear/images/labels.txt` contains **1136 rows**, each
  `images/<file>.jpg<TAB>clear`.
- `summary.json` records the original `*_sem_obstaculos` export source plus
  the merged SSIM 0.95 final-second source.
- `images.zip` and `clear.zip` were rebuilt from the current files.

## Sources

The base clear set came from `data/exports/*_sem_obstaculos/<same-name>.mp4`,
deduplicated with SSIM threshold `0.97` and max gap `0.5 s`, producing 619
images from 16 clips.

The merged supplement is
`data/derived/clear_ssim095_last1s_until_112/`, generated by
`scripts/make_clear_ssim_last1s_dataset.py` from the last second of the first
112 uploaded videos. It uses SSIM threshold `0.95` and labels every retained
image as clear (`0 2`).

## Repair note

On 2026-06-11, `clear_ssim095_last1s_until_112` had 517 JPG files but 529 label
rows. Twelve rows referenced images that were no longer present on disk, all
from uploaded clips 29 and 30. The labels were filtered to the actual kept
files:

- `labels.txt`: 517 rows
- `labels_with_source.tsv`: 517 rows
- `class_map.json total_images`: 517
- `labels_repaired.removed_label_rows_without_image`: 12

Those 517 kept images were then copied into `data/derived/clear/images/`.

## Metric variants (2026-06-12)

`scripts/make_clear_metric_last1s_dataset.py` reruns the same last-1-second
selection pipeline (keep a frame when similarity to the last kept frame drops
below the threshold; forced keep at 0.5 s gap; first/last always kept) with a
pluggable similarity metric instead of SSIM. None of these have been merged
into `data/derived/clear/images/`; they are side-by-side alternatives to the
SSIM 0.95 supplement for comparison.

| metric | similarity definition | threshold | output dir | images |
|---|---|---|---|---|
| `dinov2` | cosine of DINOv2 ViT-S/14 CLS embeddings (224x224) | 0.98 | `clear_dinov2098_last1s_until_112/` | 444 |
| `lpips` | 1 − LPIPS(AlexNet) distance (256x144) | 0.95 | `clear_lpips095_last1s_until_112/` | 476 |
| `vif` | sewar `vifp` on 160x90 grayscale (same input as SSIM) | 0.70 | `clear_vif070_last1s_until_112/` | 1386 |

All three ran over the same first 112 uploaded clips with 112/112 clips ok and
JPG count == label rows (no repair needed). For reference, SSIM 0.95 kept 517.
Thresholds were calibrated on a 2-clip sample: DINOv2 consecutive-frame cosine
sits ~0.975–0.985 and LPIPS similarity ~0.93–0.97, so SSIM-like thresholds
transfer; VIF between consecutive frames is only 0.60–0.93, so 0.95 keeps
every frame and the default was lowered to 0.70 (still ~2.7x more images than
the other metrics — VIF is the most motion-sensitive of the four).

The TSV column `ssim_prev` from the SSIM script is named `metric_prev` here.
Requires `torch`, `lpips`, `sewar`; DINOv2 weights load via `torch.hub`
(`facebookresearch/dinov2`, `dinov2_vits14`).

## Consumers

`scripts/make_kaggle_two_head_dataset.py` copies every `*.jpg` from
`data/derived/clear/images/` into `kaggle_two_head_dataset/dataset/` with a
`clear__` prefix and writes `obstacle_class=0`, `deviation_class=2`.
